# Task Bar Hero Anime

План по созданию idle/clicker-игры в духе Task Bar Hero, но с anime-inspired персонажами. Сначала делаем маленький браузерный прототип, потом переносим его в desktop overlay над панелью задач Windows.

## Core Loop

Герой атакует монстра -> монстр теряет HP -> игрок помогает кликами -> монстр умирает -> игрок получает золото -> покупает улучшения -> герой становится сильнее -> открываются более сложные монстры и зоны.

## How to Run

Открой `index.html` в браузере. Проект пока не требует сборщика, npm-зависимостей или сервера.

## MVP: Browser Playable Prototype

MVP считается готовым, когда:

- [x] Есть `index.html`, `styles.css` и JS-код в `src/`.
- [x] На экране видны герой, монстр, HP монстра, золото, DPS и урон клика.
- [x] Герой автоматически атакует монстра раз в секунду.
- [x] Игрок может кликать по монстру и наносить дополнительный урон.
- [x] После смерти монстра начисляется золото и появляется следующий монстр.
- [x] Есть хотя бы одно улучшение, покупаемое за золото.
- [x] Прогресс сохраняется в `localStorage` и восстанавливается после перезагрузки страницы.

## Initial Balance

Стартовые формулы можно менять после теста, но для первого прототипа используем простые правила:

- HP монстра: `10 * 1.18 ^ (monsterLevel - 1)`.
- Награда золотом: `5 * 1.12 ^ (monsterLevel - 1)`.
- Урон клика: `1 + clickUpgradeLevel`.
- DPS героя: `1 + swordLevel * 0.75`.
- Стоимость улучшения меча: `10 * 1.25 ^ swordLevel`.
- Стоимость улучшения клика: `8 * 1.25 ^ clickUpgradeLevel`.
